# 事实簿功能分析 + 重建计划

## 一、当前五个子标签的功能和现状

### 1. 设定 (canon_facts)
- **数据**：`canon_facts` 表，按 category 分类（角色/世界观/规则/关系/事件）
- **用途**：存储不可违反的硬规则，注入正文 prompt 的 canonFactsContext
- **操作**：手动增删改 + 大纲提取 + 卷级聚合
- **问题**：
  - 卷筛选栏最近才加上，之前不显示
  - "大纲提取"按钮只提取基础事实，不提取 `volume_summaries`
  - 不区分全局/卷级事实的视觉差异

### 2. 时间线 (story_timeline)
- **数据**：`story_timeline` 表，记录官提取的事件
- **用途**：追踪故事中的时间推进，检测跳跃
- **操作**：只读（删除），生成时自动写入
- **问题**：信息密度低，只有事件描述+时间标签+位置+角色。对于写作辅助价值有限。

### 3. 伏笔 (foreshadowing_registry)
- **数据**：`foreshadowing_registry` 表，记录官提取
- **用途**：回收率统计、逾期检测、跨卷追踪
- **操作**：手动标记回收/激活 + 智能匹配建议
- **问题**：依赖记录官 AI 的输出质量，自动提取准确度一般

### 4. 动态 (chapter_fact_deltas) — 新增
- **数据**：`chapter_fact_deltas` 表
- **用途**：每章的情感/位置/关系变化
- **操作**：只读

### 5. 冲突 (conflict_facts)
- **数据**：`conflict_facts` 表
- **用途**：设定矛盾检测
- **操作**：只读 + 跨卷检查

## 二、评估：哪些该删、哪些该留

| 标签 | 对写作价值 | 用户可操作 | 建议 |
|------|-----------|-----------|------|
| 设定 | ⭐⭐⭐ 高 | 增删改 | **保留**，是核心 |
| 时间线 | ⭐ 低 | 仅删 | **删除**，对写作帮助小，且记录官自动写入不由用户控制 |
| 伏笔 | ⭐⭐ 中 | 标记回收 | **保留但简化**，合并到设定中作为 filters |
| 动态 | ⭐⭐ 中 | 只读 | **保留**，有价值但只需简要显示 |
| 冲突 | ⭐⭐ 中 | 只读 | **保留** |

**时间线应删除**：记录官自动写入、用户无法操作、信息密度低、对写作决策帮助不大。故事时间信息可以通过进度上下文体现。

## 三、重建方案：四标签精简

```
事实簿
├─ 设定 (canon_facts)     ← 核心，保留
├─ 伏笔 (foreshadowing)   ← 保留，从"卷章展开"改为按卷展开的卡片列表
├─ 动态 (deltas)          ← 保留
└─ 冲突 (conflicts)       ← 保留
```

删除时间线标签，伏笔的跨卷追踪和回收率统计已足够替代其功能。

### 设定标签增强
- 卷筛选栏：`[全部] [全局(vol=0)] [第1卷] [第2卷]...`
- 事实卡片显示 `volume_number` 标签
- "从大纲提取" → 改名为"重新提取全部事实"，功能增强为：
  1. 清空所有 `source='outline'` 的事实
  2. 调 CANON_EXTRACTION 提取基础事实（角色/设定/规则/关系/事件）
  3. 额外调一次 extractVolumeSummaries 提取分卷概要 → volume_summary facts
  4. 一次性完成，提示"已提取 X 条事实 + Y 条卷概要"

### 伏笔标签简化
- 保留回收率统计条 + 卷筛选
- 保留建议回收区域
- 去掉"按卷章展开"的深嵌套，改为按卷展开的平铺卡片

### 动态和冲突
- 保持不变，功能完整
