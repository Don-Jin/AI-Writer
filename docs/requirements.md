# 需求规格说明

## 项目概述

AI 小说写作软件 —— 一款 Windows 桌面应用，利用 DeepSeek AI 辅助用户进行长篇小说创作。用户可以导入现有小说让 AI 学习其写作风格，生成「风格库」模板，在创作新小说时套用不同风格。

## 目标用户

- 网络小说作者
- 创意写作爱好者
- 需要 AI 辅助激发灵感的写作者

## 功能需求

### FR-1 风格库管理

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| FR-1.1 | 导入小说 | 支持上传 TXT/DOCX/PDF 文件，或粘贴文本 | P0 |
| FR-1.2 | AI 风格提取 | 调用 DeepSeek API 分析小说，提取写作风格、语言特点、修辞手法、氛围基调 | P0 |
| FR-1.3 | 保存风格库 | 将提取结果保存为可命名的风格模板 | P0 |
| FR-1.4 | 风格库列表 | 展示所有已创建的风格库，可查看详情、删除 | P0 |

### FR-2 小说项目管理

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| FR-2.1 | 新建项目 | 输入书名、简介，选择风格库（主风格+可选辅风格） | P0 |
| FR-2.2 | 项目列表 | 首页展示所有项目，显示书名、创建时间、进度状态 | P0 |
| FR-2.3 | 项目管理 | 支持项目重命名、删除 | P1 |

### FR-3 三阶段写作流程

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| FR-3.1 | 大纲生成 | AI 根据书名+简介+风格生成完整故事大纲（主线、人物、背景、冲突、结局） | P0 |
| FR-3.2 | 大纲编辑 | 用户可修改大纲内容 | P0 |
| FR-3.3 | 细纲生成 | AI 基于大纲生成逐章细纲（章节概要、出场人物、关键事件、字数预估） | P0 |
| FR-3.4 | 细纲编辑 | 用户可调整章节顺序、增删章节、修改内容 | P0 |
| FR-3.5 | 逐章生成 | AI 基于大纲+细纲+风格库逐章生成正文 | P0 |
| FR-3.6 | 章节编辑 | 每章生成后可修改，支持重新生成 | P0 |
| FR-3.7 | 上下文管理 | 自动维护人物状态表和情节摘要，确保长篇一致性 | P0 |

### FR-4 写作编辑器

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| FR-4.1 | 文本编辑 | 所见即所得编辑器，支持富文本/纯文本 | P0 |
| FR-4.2 | 章节导航 | 侧边栏展示章节列表，点击切换 | P0 |
| FR-4.3 | 风格标签 | 编辑器显示当前使用的风格库标签 | P1 |

### FR-5 导出功能

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| FR-5.1 | 格式支持 | 支持导出 TXT、DOCX（Word）、PDF | P0 |
| FR-5.2 | 范围选择 | 支持导出单章或整本书 | P1 |
| FR-5.3 | 目录生成 | 自动生成章节目录页 | P1 |

### FR-6 系统设置

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| FR-6.1 | API Key 配置 | 配置 DeepSeek API Key | P0 |
| FR-6.2 | 数据存储路径 | 查看/修改本地数据存储位置 | P2 |

## 非功能需求

- **性能**：章节生成响应时间 < 30 秒（取决于 DeepSeek API）
- **数据安全**：所有数据本地存储，不上传至任何第三方服务器（除 DeepSeek API 调用外）
- **可用性**：界面简洁直观，无需编程知识即可使用
- **兼容性**：支持 Windows 10 及以上系统
